# Índice de skills

Todos los especialistas de ASDT de un vistazo, en orden de pipeline. Cada uno es invocable de forma independiente con su comando — usá esta tabla para elegir el especialista correcto cuando solo necesitás una perspectiva.

| Especialista | Comando | Propósito |
|--------------|---------|-----------|
| [Researcher](/es/docs/specialists/researcher) | `/asdt-researcher` | Descubrimiento pre-PM — ideación divergente, escaneo de factibilidad y una única dirección recomendada |
| [Product Manager](/es/docs/specialists/pm) | `/asdt-pm` | Convierte peticiones crudas en backlog entries estructurados con historias de usuario y límites de alcance |
| [Architect](/es/docs/specialists/architect) | `/asdt-architect` | Decisiones de arquitectura, ADRs y diseño de sistemas con trade-offs explícitos |
| [Developer](/es/docs/specialists/developer) | `/asdt-developer` | Implementación — exploración, diseño técnico y código de producción |
| [QA](/es/docs/specialists/qa) | `/asdt-qa` | Estrategia de testing, validación de criterios de aceptación y quality gates |
| [Security](/es/docs/specialists/security) | `/asdt-security` | Modelado de amenazas, análisis de superficie de ataque y checklists de hardening |
| [UX/UI](/es/docs/specialists/ux-ui) | `/asdt-ux-ui` | Flujos de usuario, arquitectura de información y specs de componentes |

Cuando el trabajo abarca múltiples perspectivas, salteá la tabla y ejecutá `/asdt <descripción del feature>` — el orquestador clasifica el cambio y lo enruta por los especialistas correctos en el orden correcto.
