# Skills Index

Every ASDT specialist at a glance, in pipeline order. Each one is independently invocable by its command — use this table to pick the right specialist when you only need one perspective.

| Specialist | Command | Purpose |
|------------|---------|---------|
| [Researcher](/docs/specialists/researcher) | `/asdt-researcher` | Pre-PM discovery — divergent ideation, feasibility scanning, and a single recommended direction |
| [Product Manager](/docs/specialists/pm) | `/asdt-pm` | Turns raw requests into structured backlog entries with user stories and scope boundaries |
| [Architect](/docs/specialists/architect) | `/asdt-architect` | Architecture decisions, ADRs, and system design with explicit trade-offs |
| [Developer](/docs/specialists/developer) | `/asdt-developer` | Implementation — exploration, technical design, and production code |
| [QA](/docs/specialists/qa) | `/asdt-qa` | Test strategy, acceptance criteria validation, and quality gates |
| [Security](/docs/specialists/security) | `/asdt-security` | Threat modeling, attack-surface analysis, and hardening checklists |
| [UX/UI](/docs/specialists/ux-ui) | `/asdt-ux-ui` | User flows, information architecture, and component specs |

When the work spans multiple perspectives, skip the table and run `/asdt <feature description>` — the orchestrator classifies the change and routes it through the right specialists in the right order.
